📋 Key Relationships

🏗️ Core Request Structure

HRRequests (1) → HRRequestDetails (Many): Parent-child relationship for batch requests
HRRequestDetails (Many) → RequestTypes (1): Defines request type
HRRequestDetails (Many) → RequestStatuses (1): Tracks request status

📝 Request Type Extensions

HRRequestDetails (1) → PromotionRequestDetails (0..1): Position changes
HRRequestDetails (1) → LayoffRequestDetails (0..1): Layoff information
HRRequestDetails (1) → TerminationRequestDetails (0..1): Termination details
HRRequestDetails (1) → ReturnToWorkRequestDetails (0..1): Return to work

⚙️ Access Features

HRRequestDetails (1) → ITDetails (0..1): IT access requirements
ITDetails (1) → ApplicationRequests (Many): Application access requests
ApplicationRequests (Many) → Applications (1): Application catalog
HRRequestDetails (1) → CreditCardDetails (0..1): Credit card requirements
HRRequestDetails (1) → VehicleDetails (0..1): Vehicle requirements

📧 Notifications

HRRequests (1) → NotificationQueue (Many): Generates notifications
EmailTemplates (1) → NotificationQueue (Many): Template-based emails
